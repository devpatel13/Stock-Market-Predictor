# Stock Market Predictor 📈

An end-to-end LLM-based financial analysis pipeline. This project scrapes real-time stock data, uses a "Teacher" LLM (Llama-3-70B) to generate high-quality reasoning and labels, and then distills this knowledge by fine-tuning a smaller, efficient "Student" LLM (Llama-3-8B) for market sentiment analysis.

## 🛠️ Tech Stack
* **Language:** Python 3.10+
* **Data Acquisition:** `yfinance`, `pandas`
* **LLM APIs:** Groq Cloud (Llama-3-70B-Versatile)
* **Fine-Tuning Frameworks:** PyTorch, Hugging Face Transformers, PEFT (LoRA), BitsAndBytes (4-bit quantization), Unsloth
* **Environment:** Kaggle Notebooks (GPU T4 x2 or P100 recommended)

---

## 📋 Prerequisites
Before running the code on Kaggle, ensure you have the following accounts and keys:

1.  **Kaggle Account:** Required to run the notebooks with GPU acceleration.
2.  **Groq Cloud API Key:**
    * **Why?** Used in `fetch_training_data.py` to access the Llama-3-70B "Teacher" model for labeling data.
    * **Get it here:** [Groq Console](https://console.groq.com/)
3.  **Hugging Face Access Token:**
    * **Why?** Required in `teacher_training.py` to download gated models (like Llama 3) and tokenizers.
    * **Get it here:** [Hugging Face Settings](https://huggingface.co/settings/tokens)
    * *Note:* You must request access to the Llama 3 model on its Hugging Face model card page first.

## 🚀 Setup & Execution Guide (Kaggle)

### Step 0: Environment Setup
1.  Create a new Notebook on Kaggle.
2.  In the right sidebar, verify **Accelerator** is set to **GPU T4 x2** (or better).
3.  **Set Secrets:**
    * Go to "Add-ons" -> "Secrets" in the top menu.
    * Add a new secret labeled `GROQ_API_KEY` with your Groq key.
    * Add a new secret labeled `HF_TOKEN` with your Hugging Face token.

### Step 1: Fetch Financial Data
* **File:** `fetch_news.py`
* **Action:** Copy the code into a cell and run it.
* **Function:**
    * Fetches the last 7 days of OHLCV data and recent news for defined tickers (e.g., `RELIANCE.NS`, `ZOMATO.NS`).
    * Generates a formatted text file `scraped_finance_data.txt` ready for LLM consumption.

### Step 2: Teacher Labeling (Data Distillation)
* **File:** `fetch_training_data.py`
* **Preparation:** Upload your raw dataset (e.g., `nifty50_train_data.jsonl`) to the Kaggle Input directory.
* **Action:** Run the script.
* **Function:**
    * Reads raw financial text.
    * Sends it to the Groq API (Llama-70B) with a Chain-of-Thought prompt.
    * Generates structured labels (Sentiment, Confidence, Decision: BUY/SELL/HOLD).
    * Saves the labeled dataset as `training_data_chatml.jsonl`.

### Step 3: Fine-Tuning the Student Model
* **File:** `teacher_training.py`
* **Action:** Run the script.
* **Function:**
    * Loads the labeled data from Step 2.
    * Downloads the base model (`unsloth/meta-llama-3.1-8b-bnb-4bit`).
    * Applies LoRA (Low-Rank Adaptation) for efficient training.
    * Fine-tunes the model to mimic the Teacher's reasoning.
    * Saves the final adapter to `./output-llm-finetune`.

## 📂 Project Structure

| File Name | Description |
| :--- | :--- |
| **`fetch_news.py`** | Utility to scrape live stock history and news headlines using Yahoo Finance. Formats data for LLM context. |
| **`fetch_training_data.py`** | The "Teacher" pipeline. Uses Groq's API to analyze raw text and create a high-quality, labeled dataset (Distillation). |
| **`teacher_training.py`** | The "Student" training script. Fine-tunes a quantized Llama 3 8B model using LoRA on the data generated by the Teacher. |

## ⚠️ Troubleshooting
* **Rate Limits (429):** If `fetch_training_data.py` stops due to API limits, the script is designed to save progress. Simply wait a few minutes and re-run; it will resume where it left off.
* **CUDA/GPU Errors:** Ensure you have selected a GPU accelerator in your Kaggle notebook settings. The training script requires a GPU.
